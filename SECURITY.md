# 安全策略 / Security Policy

## 支持版本

| 版本 | 支持状态 |
|------|----------|
| 2.0.x | ✅ 活跃支持 |
| < 2.0 | ❌ 不再支持 |

## 报告漏洞

如发现安全漏洞，**请不要公开提交 Issue**。请通过以下方式私下报告：

- 若仓库托管在 **GitHub**：在仓库页面 **Security → Report a vulnerability** 提交（仓库需启用 *Private vulnerability reporting*；未启用时由维护者在 *Settings → Security* 中打开）。
- 其他托管方式或无法使用上述入口时：通过维护者提供的私密渠道（例如安全联系邮箱）联系；**勿在 Issue / 讨论区贴出利用细节或 PoC**。

我们会尽快确认并处理，通常在 48 小时内回复。

## 安全最佳实践

- 所有代码执行默认在沙箱中运行
- C 编译产物支持地址消毒器 (AddressSanitizer) 集成
- 运行时不执行 `eval` / `Function()` 动态代码
- 模块导入受白名单策略约束

## 披露政策

漏洞修复后，我们会在 CHANGELOG 中记录，并在发布说明中致谢报告者（需本人同意）。
